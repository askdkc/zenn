---
title: ログ調査――巨大ログを丸呑みしない
---
ある朝、社内アプリで障害が出た。

営業部の画面が固まり、問い合わせが情報システム部に流れ込んできた。高村はログファイルの場所を開いた。

```text
app-2024-04-01.log
size: 4.8GB
```

ミナは一瞬で嫌な顔をした。

「ロ勹″ｶゝ……廾イズ次第τ″ﾚよEmacsﾅﾆ″`⊂辛ﾚヽωﾅﾆ″ょぅ。」

高村は一秒止まった。

「翻訳して。」

「ログか……サイズ次第ではEmacsだと辛いんだよぅ、です。」

黒瀬が聞いた。

「Emacsでも巨大ファイルを開ける方法はあるの？」

「あります。VLFっていう巨大ファイルを見るためのパッケージがあります。全部を一気に読むんじゃなくて、分割して少しずつ見る感じです。」

「じゃあ、それで開けばいい？」

「目的次第です。今回はエラー行だけ欲しいので、全部見る必要がありません。まず `rg` で絞ります。」

「全部見る必要がないのに巨大ファイルを開くのは？」

「冷蔵庫ごと作業机に乗せるみたいなものです。必要な材料だけ出します。」

ミナはEmacs内の右上、eatモードのターミナルで打った。

```sh
rg -n "ERROR|WARN" app-2024-04-01.log > /tmp/app-errors.log
```

高村はコマンドを見て、すぐに聞いた。

「`rg -n` って何？」

ミナはコマンドを一つずつ区切って説明した。

```text
rg
  ripgrep という検索コマンド。
  大きなテキストファイルから、条件に合う行を高速に探す。

-n
  行番号を表示するオプション。
  後で「何行目にあったか」を追える。

"ERROR|WARN"
  探したい文字列の条件。
  ERROR または WARN を含む行を探す。
  ここでの | は「または」の意味。

app-2024-04-01.log
  調べるログファイル。

>
  出力先を変える記号。
  画面に流す代わりに、右側のファイルへ保存する。

/tmp/app-errors.log
  抽出結果を保存する小さなファイル。
```

「つまり、巨大ログを直接読むんじゃなくて、ERRORかWARNを含む行だけを別ファイルに抜き出してるんだね。」

「そうです。全部を飲むんじゃなくて、まず必要そうな行のリストを作ります。」

## 何をしているか

巨大ログ全体をEmacsで直接開かず、`rg`で `ERROR` と `WARN` を含む行だけを抽出して、小さなファイルに保存している。

```text
巨大ログ
  ↓ rg -n "ERROR|WARN"
エラー行・警告行のリスト
  ↓ Emacs
読みやすい調査対象
```

## 失敗例

```sh
cat app-2024-04-01.log
```

巨大ログをそのままeatに流すと、端末バッファが重くなり、Emacsの操作も重くなる。  
EmacsにはVLFのような巨大ファイルを見るための道具もあるが、通常の編集と同じ感覚で扱うと厳しい。今回の目的はエラー行の確認なので、見る前に絞るのが先である。

ミナは `/tmp/app-errors.log` をEmacsで開いた。ようやく、人間が読めるサイズになった。

```text
2024-04-01T09:00:03 ERROR request_id=R003 user=122 code=E502 message="upstream timeout"
2024-04-01T09:00:04 WARN request_id=R004 code=W201 message="retry"
```

## `occur`でエラー行を一覧する

```elisp
(occur "ERROR")
```

`occur` は、条件に合う行を一覧する。  
ログの中から、さらに `ERROR` を含む行だけを見る。

## request_idで文脈を追う

エラー行だけでは原因が分からない。

```text
ERROR request_id=R003 code=E502
```

同じ `request_id` の前後を見る必要がある。

```elisp
(occur "request_id=R003")
```

ログ調査は、検索ではなく、物語を復元する作業である。

## ミナのメモ

ミナは右下のメモバッファに、ここまでの変換を整理した。

```text
現実:
  4.8GBの巨大ログがある。

判断:
  全部をEmacsで丸呑みしない。
  エラー行だけ欲しいならrgで絞る。
  全体を目視したいならVLFを使う。

抽象化:
  巨大ログ
    ↓
  エラー行のリスト
    ↓
  request_idごとの物語

使った道具:
  rg
  eat
  occur
  VLF
```
