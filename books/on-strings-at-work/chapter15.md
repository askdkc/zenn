## 14.1 問題をリストにする手順

```text
1. 現実の困りごとを言葉にする
2. 何が混ざっているかを分ける
3. 必要なものを行のリストにする
4. 行を名前と値に分ける
5. 業務上の意味を与える
6. 小さな関数で扱う
7. 人間に戻す
```


## 14.2.1 コマンドと記号の最小メモ

```text
rg
  ripgrep。大きなテキストから条件に合う行を高速に探すコマンド。

-n
  行番号を表示するオプション。

"ERROR|WARN"
  ERROR または WARN を探す条件。
  | は「または」。

>
  コマンドの出力を、画面ではなくファイルへ保存する。

SELECT *
  SQLで、すべての列を見る。

WHERE
  条件に合う行だけを見る。

GROUP BY
  同じ値ごとにまとめる。

HAVING
  GROUP BYでまとめた後の結果に条件をかける。
```

## 14.3.1 この本で自作した小さな関数

```text
split-on-char
  Common Lisp標準にはない。
  学習用に作った、指定文字で文字列を分割する関数。

ensure-list
  Common Lisp標準にはない。
  nil、list、vector、単独の値をlistとして扱いやすくするユーティリティ。

parse-amount-field
  金額文字列を整数へ変換する。
  成功・失敗・元の値を区別して返すため、0円をエラー扱いしない。
```

## 14.4 Common Lisp系

```text
string-trim
  前後の文字を削る。

position
  文字の位置を探す。

subseq
  一部を取り出す。

parse-integer
  文字列を整数に変える。

handler-case
  エラーを受け止める。
```

## 14.5 練習問題

1. 自分の仕事で「文字列地獄」になっているものを一つ挙げる。
2. その中に何が混ざっているかを言葉にする。
3. 必要な行のリストを手で作る。
4. 一行を名前と値に分ける。
5. それを plist 風に書く。
6. 人間が判断すべきものと、機械に任せてよいものを分ける。
