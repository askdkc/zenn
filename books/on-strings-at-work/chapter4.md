---
title: 最初の文字列地獄
---
自己紹介が一段落したところで、黒瀬は一枚の紙をミナに渡した。

```text
インターン初日 課題1:
営業部から来た申請メールを整理し、週次でDB投入できる形にすること。
```

ミナは少しだけ安心した。

「メール整理なら、最初の仕事としては普通かもしれない。」

そう思ったのは、最初の五分だけだった。

黒瀬が説明した。

「営業部から申請メールが来る。今は担当者が一件ずつ開いて、必要な項目をExcelへコピペしている。金曜にそのExcelをCSVにして、社内DBへ流し込む。」

「毎週ですか？」

「毎週。」

「件数は？」

「少ない週で百件。多いと千件近い。」

ミナは口元を引きつらせた。

「うげっ、それ手でやるんすか。ダルぃなぁ……。」

黒瀬は声を出して笑った。

「ははははっ。いや、その反応は正しい。インターン初日の仕事としては、業務の流れを知るには悪くないんだけどね。」

「流れを見る前に心が死にそうですけど。」

「そこまで言う？」

最初のメールは、まだ読めた。

```text
件名: 申請 A-1029
氏名: 山田　太郎
部署: 営業一課
金額: 12,000円
希望日: 2024/04/01
備考: 急ぎ
```

次のメールは、少し違った。

```text
申請番号 A-1030
名前： 佐藤 花子
部門：営業2課
金額：12000
希望日：2024年4月2日
```

さらに別のメールは、長い挨拶文の中に必要情報が埋もれていた。

```text
お世話になっております。
以下の件について確認をお願いいたします。

注文番号: A-1031
担当: 鈴木
部署: 営業部
申請金額: 9,000円
納期希望: 2024-04-03

何卒よろしくお願いいたします。
```

ミナは画面を見た。

一件ずつ読めば分かる。だが、いきなりExcelに貼るには表記が揺れすぎている。`氏名` と `名前` と `担当`。`部署` と `部門` と `所属`。`金額` と `申請金額`。金額も `12,000円`、`12000`、`9,000円` が混ざっている。

「これ、最初からExcelに放り込むより、一回テキストとして集めて観察した方がいいですね。」

高村が身を乗り出した。

「お、面白い視点だね？ 詳しく聞かせて。」

「メール本文をそのままExcelに入れると、あとで列に分ける時に地獄です。まずテキストとして集めます。そこで、どんな揺れがあるかを見ます。その日の最後にExcelへまとめる。金曜にCSV化してDBへ持っていく。順番を分けた方が安全です。」

高村は頷いた。

「つまり、Excelには最初からメールを放り込むんじゃなくて、テキストで汚れを見て、整理した後に置きたいってこと？」

「そうです。Excelをゴミ箱にしないために、手前で文字列を見る感じです。」

ミナはホワイトボードに書いた。

```text
困っていること:
  メールの書き方が揺れている

混ざっているもの:
  申請番号
  名前
  部署
  金額
  希望日
  備考

揺れている表現:
  氏名 / 名前 / 担当
  部署 / 部門 / 所属
  金額 / 申請金額
```

黒瀬が言った。

「こうやって書くと、ただメールが多いんじゃなくて、何が混ざっているかが見えるな。」

「はい。まず問題を言葉にします。言葉にできると、リストにできます。」

ミナはメールの一部を指した。

```text
氏名: 山田　太郎
部署: 営業一課
金額: 12,000円
```

「これは文章に見えますけど、項目のリストです。」

```text
氏名
部署
金額
```

「さらに分けると、名前と値のペアです。」

```text
氏名 -> 山田　太郎
部署 -> 営業一課
金額 -> 12,000円
```

「ここまで来ると、Lispでも扱えます。」

高村が画面を見た。

「文章のふりをした申請データか。」

「そうです。人間には文章、DBにはレコード。その間にあるのが、リストです。」

## ミナのメモ

ミナは右下のメモバッファに、ここまでの変換を整理した。

```text
現実:
  営業メールが多く、表記も揺れている。

言語化:
  氏名/名前/担当、部署/部門/所属、金額/申請金額が混ざっている。

リスト化:
  長いメール
    ↓
  必要項目のリスト
    ↓
  名前と値のペア

次へ:
  Emacsで必要行を残し、不要行を消す。
```
