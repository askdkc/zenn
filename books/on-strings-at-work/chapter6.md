---
title: 空白と表記ゆれ――人間には同じでも機械には違う
---
三日目、ミナは社員名簿の整備を頼まれた。

ファイルを開くと、同じ人名が微妙に違う形で並んでいた。

```text
山田　太郎
山田 太郎
 山田 太郎 
山田太郎
YAMADA TARO
```

黒瀬が言った。

「人間が見れば山田さんだと分かる。システムでも同じ扱いにできる？」

ミナは言った。

「段階を分けます。空白を揃えることと、ローマ字を日本語名に寄せることは別問題です。」

ミナは名簿の先頭を見ながら、ため息をついた。

「うひぃ、見た目は同じ山田さんなのに、文字列としては別人扱い。こういうのが一番だるぃよぅ……。」

まずは、前後の余計な空白だけを削る。人間が手で消すには地味すぎるが、機械に任せるにはちょうどいい。

```lisp
(string-trim " " " 山田 太郎 ")
```

## 何をしているか

`string-trim` は、文字列の前後にある指定文字を削る。
ここでは半角スペースを削っている。

## 入力例

```lisp
" 山田 太郎 "
```

## 出力例

```lisp
"山田 太郎"
```

## 失敗例

```lisp
(string-trim " " "山田　太郎")
;; => "山田　太郎"
```

全角スペースは半角スペースとは別の文字なので削れない。

## リファクタリング

全角スペースを半角スペースへ置換してから、前後空白を削る。

```lisp
(defun replace-char (from to string)
  (map 'string
       (lambda (ch)
         (if (char= ch from) to ch))
       string))

;; `#\Ideographic_Space` は処理系依存です。SBCLなどUnicode文字名を扱える処理系を前提にしています。
;; 最近はSBCLが標準的に使われるのでSBCL前提で進めます。
(defun normalize-name-space (name)
  (string-trim " "
               (replace-char #\Ideographic_Space #\Space name)))
```

## 入力例

```lisp
(normalize-name-space " 山田　太郎 ")
```

## 出力例

```lisp
"山田 太郎"
```

## 失敗例

```lisp
(normalize-name-space "YAMADA TARO")
;; => "YAMADA TARO"
```

高村が聞いた。

「ローマ字の `YAMADA TARO` も、山田太郎に寄せられない？」

ミナは少し考えてから首を横に振った。

「今回は無視します。ローマ字を日本語名に寄せるには、別の辞書や名寄せルールが必要です。山田太郎さんが一人とは限らないし、Yamada Taroがどの山田太郎なのかは文字列だけでは決められません。」

「やると大変になる？」

「大変というより、勝手にやると事故ります。今回は空白の揺れだけ直して、ローマ字名寄せは別タスクにします。」

そのあと、ミナは社員名簿のうち、空白の揺れだけを整えた版を作った。ローマ字名寄せには手を出さない。そこは人事側の判断が必要だからだ。

高村は結果を見て言った。

「同じ山田さんが、三人いるように見えていたのが、一人に見えるようになった。」

「空白だけなら、機械で直せます。でも、人の同一性は文字列だけでは決めません。」

## ミナのメモ

ミナは右下のメモバッファに、ここまでの変換を整理した。

```text
現実:
  見た目が似た名前が、文字列としては別物になっている。

言語化:
  前後空白
  全角スペース
  半角スペース
  名寄せ

処理:
  normalize-name-space

判断:
  空白の揺れは機械で直す。
  ローマ字名寄せは人間の判断に回す。
```
